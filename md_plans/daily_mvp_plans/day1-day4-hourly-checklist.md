# 四天总控执行清单（按小时打勾版）

> 适用范围：上位机（GEC-6818）+ 下位机（portable_code 参考）并行推进  
> 使用方式：每完成一项打勾；若未完成，必须填写“阻塞原因 + 回退动作”

---

## 全局规则（每天都执行）

- [ ] 每天 09:00 前确认当天 MVP 成功标准（只保留 3~5 个硬目标）
- [ ] 每天 12:00 做中期检查（进度、风险、是否降级）
- [ ] 每天 18:00 做收口验收（功能演示 + 日志 + 问题清单）
- [ ] 每天至少保留 1 个可演示版本，不在晚间做高风险重构
- [ ] 每天更新文档：已完成 / 未完成 / 次日转移项

---

## Day 1（上位机调试增强 + 下位机本地感知）

### 09:00-10:00 启动与环境确认

- [ ] 明确 Day1 MVP：天气时间可显示、历史首帧链路可落地、守护可触发
- [ ] 上位机启动链路验证：开机动画 -> 主界面 -> 触控响应
- [ ] 下位机传感器链路验证：超声波、温湿度、蜂鸣器
- 产出物：
  - [ ] 启动日志截图/文本
  - [ ] 当日风险清单 v1

### 10:00-12:00 上位机天气时间问题修复主攻

- [ ] 排查 `main.c` 与 `weather_time_thread` 参数与线程逻辑一致性
- [ ] 时间每秒刷新可见
- [ ] 天气不可用时做“缓存/暂无数据”降级
- 产出物：
  - [ ] 天气时间线程调试记录
  - [ ] 可稳定显示的演示结果

### 13:30-15:30 历史记录 MVP（首帧缩略图 -> 回放）

- [ ] 录像时保存首帧 bmp 与录像主文件绑定
- [ ] 历史页可选择记录并触发对应回放
- [ ] 失败时至少保留“首帧显示 + 对应文件路径”
- 产出物：
  - [ ] 历史索引规则说明（时间戳/命名）
  - [ ] 最小回放链路截图/日志

### 15:30-17:30 守护进程 MVP（循环录像）

- [ ] 达到 `MAX_RECORDS` 时触发删除最旧录像
- [ ] 删除后继续录制不中断
- [ ] 守护模式运行日志可观察（删除对象、时间戳）
- 产出物：
  - [ ] 删除策略记录（按修改时间）
  - [ ] 守护模式连续运行日志

### 19:00-20:00 Day1 收口验收

- [ ] 上位机：动画/触控/时间/录像/历史/守护可演示
- [ ] 下位机：采集+告警可稳定输出
- [ ] 更新 Day2 输入条件（协议字段、未解决问题）
- 未完成回退：
  - [ ] 若天气不稳，保留“时间稳定 + 天气降级显示”
  - [ ] 若历史回放未全通，保留“首帧 + 文件映射可查”
